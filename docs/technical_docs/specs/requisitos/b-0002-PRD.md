# Visão do projeto

Atualmente (12/07/2025) o projeto se encontra com algumas implementações realizadas, ainda não em produçao devido a necessidade de refatoração do front-end para aplicações por domínio.

O projeto recebeu feedbacks positivos da escola e pontos de melhorias, isso será passado nesse documento.

--- 

## Objetivos atuais

1. Refatoração e padronização
2. Criação de LIB de UI
3. Adicionar maiores diferenciais do projeto como sistema de turmas
4. Sistema freemium 

---

## Requisitos funcionais (RF)

| ID   | Requisito                            | Prioridade | Descrição                                                                                                                                                        | Observações |
| ---- | ------------------------------------ | ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------- |
| RF01 | Cadastro de usuário                  | Alta       | O sistema deve permitir criar uma conta com e-mail e senha.                                                                                                      |             |
| RF02 | Login com OAuth2                     | Alta       | O sistema deve autenticar o usuário via OAuth2.                                                                                                                  |             |
| RF03 | CRUD Questões                         | Alta       | O sistema deve ser capaz de realizar CRUD de questões.                                                                                                           |             |
| RF04 | CRUD de conteúdos                    | Alta       | O sistema deve ser capaz de realizar CRUD de conteúdos.                                                                                                          |             |
| RF05 | CRUD de Matéria                      | Média      | O sistema deve ser capaz de realizar CRUD de matérias.                                                                                                           |             |
| RF06 | CRUD de Atividades                   | Alta       | O sistema deve ser capaz de realizar CRUD de atividades.                                                                                                         |             |
| RF07 | Sistema de campeonatos               | Baixa      | O sistema deve gerenciar campeonatos, consistindo em uma coletânea de questões que o usuário deverá acertar para competir com amigos e outros usuários.          |             |
| RF08 | Sistema de Badges                    | Baixa      | O sistema deve gerenciar badges na plataforma para recompensar alunos.                                                                                           |             |
| RF09 | Sistema de "termo" matemático        | Baixa      | Sistema parecido com o Nerdle para gerar desafios diários de matemática aos usuários.                                                                            |             |
| RF10 | Sistema de quizzes matemáticos       | Baixa      | O sistema deve gerenciar quizzes matemáticos, que aparecerão no primeiro acesso diário do usuário.                                                               |             |
| RF11 | Alta acessibilidade                  | Alta       | O sistema deve possuir alta acessibilidade para todos os tipos de usuários.                                                                                      |             |
| RF12 | Sistema Freemium                     | Média      | O sistema deve gerenciar usuários free e premium, garantindo segurança de dados pessoais.                                                                        |             |
| RF13 | Sistema de roadmap                   | Baixa      | O sistema deve apresentar um roadmap mostrando aos alunos passos para aprender determinado assunto de matemática.                                                |             |
| RF14 | Modo dislexia                        | Média      | O sistema deve oferecer métodos para melhorar a experiência de usuários com dislexia, como fontes adequadas.                                                     |             |
| RF15 | Download de conteúdos da plataforma  | Alta       | O sistema deve permitir baixar conteúdos (simulados, provas, questões, atividades, posts) em PDF.                                                                |             |
| RF16 | Ranking e sistema de pontuação       | Baixa      | O sistema deve manter ranking global e entre amigos baseado em pontuação de quizzes e campeonatos.                                                               |             |
| RF17 | Notificações e lembretes             | Média      | O sistema deve enviar notificações sobre novas atividades, desafios diários e lembretes de estudo.                                                               |             |
| RF18 | Sistema de mensagens e comunidade    | Baixa      | O sistema deve permitir mensagens entre usuários e fóruns temáticos para troca de conhecimento.                                                                  |             |
| RF19 | Área administrativa (Painel Admin)   | Alta       | O sistema deve possuir painel administrativo para gerenciar usuários, conteúdos e estatísticas.                                                                  |             |
| RF20 | Sistema de comentários e avaliações  | Média      | O sistema deve permitir que usuários comentem e avaliem conteúdos ou quizzes.                                                                                    |             |
| RF21 | Controle de tempo de estudo          | Baixa      | O sistema deve monitorar o tempo dedicado pelo aluno aos estudos e permitir definição de metas semanais.                                                         |             |
| RF22 | Multi-idioma                         | Baixa      | O sistema deve oferecer suporte a múltiplos idiomas, inicialmente português e inglês.                                                                            |             |
| RF23 | Painel de conquistas                 | Baixa      | O sistema deve exibir todas as conquistas e badges desbloqueadas pelo usuário.                                                                                   |             |
| RF24 | Controle de permissões e papéis      | Alta       | O sistema deve gerenciar diferentes níveis de acesso (aluno, professor, admin, premium).                                                                         |             |
| RF25 | Exportação de relatórios             | Baixa      | O sistema deve permitir exportar relatórios em PDF, CSV e Excel.                                                                                                 |             |
| RF26 | Integração com sistema de pagamento  | Baixa      | O sistema deve se integrar a gateways de pagamento para planos premium.                                                                                          |             |
| RF27 | Sistema de logs e auditoria          | Alta       | O sistema deve registrar todas as ações críticas (login, edições, exclusões) para fins de auditoria.                                                             |             |
| RF28 | IA Tutora Personalizada (Fine-Tuned) | Baixa      | O sistema deve contar com uma IA tutora treinada para auxiliar alunos durante estudos, oferecendo explicações, sugestões de conteúdo e feedbacks personalizados. |             |
| RF29 | Nível WCAG2.1AAA                     | Média      | O sistema deve ser capaz de passar em todos os testes da WCAG2.1AAA, para permitir maior acessibilidade do site.                                                  |             |
| RF30 | Flashcards                           | Baixa      | Sistema deve ser capaz de gerenciar flashcards, onde o usuário poderá criar, editar, remover, visualizar, responder, etc.                                         |             |
| RF31 | Algoritmo de revisão                 | Baixa      | Sistema deve possuir um algoritmo de revisão que informa ao usuário quando ele precisará revisar determinado conteúdo baseado nas suas respostas (ex.: Feynman).   |      Dependente do sistema de notificação (RF17)    |
| RF32 | Turmas | Média | Sistema deve ser capaz de permitir a cricação de turmas no sistema e seus gerenciamento pelos Admins e gerenciamento limitado para usuários da plataforma | Qualquer usuário poderá criar turmas, elas servirão como uma comunidade para aumentar o engajamento permitindo que haja um meio de intereção com outras pessoas; turmas serão gratuitas |
| RF33 | Login social (Google, Apple, Facebook)| Alta       | Permitir autenticação rápida via provedores sociais (Google, Apple, Facebook) além do OAuth2 padrão.                                                               | Complementa RF02                                |
| RF34 | Autenticação de dois fatores (2FA)    | Média       | Oferecer 2FA via SMS, e-mail e aplicativos autenticadores (TOTP) para contas com permissões sensíveis.                                                             | Requer configuração de envio de cBem-vindo à seção de especificações técnicas do projeto.ódigos (RF17)  |
| RF35 | Temporizador de estudo (Pomodoro)     | Média      | Permitir criar sessões de estudo com temporizador (Pomodoro), com pausas e histórico de sessões por usuário.                                                       | Integra com controle de tempo (RF21)            |
| RF36 | Provas e questões temporizadas        | Alta       | Permitir criação de provas/atividades temporizadas com autosave, bloqueio ao estourar o tempo e opção de temporizador por questão.                                 | Depende de sessão e auditoria (RNF03/RNF10)     |
| RF37 | Suporte e Central de Ajuda            | Alta       | Incluir sistema de suporte ao usuário: base de conhecimento, tickets, chat em tempo real e histórico de solicitações.                                              | Integra com painel administrativo (RF19)        |
| RF38 | Questão do dia via e-mail             | Média      | Enviar diariamente uma ou mais questões aleatórias por e-mail com CTA para abrir a plataforma e resolver; personalizável por assunto/nível.                        | Deve respeitar preferências de notificação (RF17) |
| RF39 | Notificações multi-canal              | Alta       | Enviar notificações por e-mail, push e SMS para lembretes de estudo, novos conteúdos, questões diárias e atualizações críticas.             |
| RF40 | Anexos | Média | Conteúdos devem possuir a possibilidade de anexar conteúdos, como: PDF, vídeos, etc | |
| RF41 | Images nas alternativas | Alta | deve ser possível adicionar imagens as alternativas devido algumas questões possuirem images | |
| RF42 | Histórico de criações | Professores, administradores, moderadores, e operadores, devem possuir historíco de criação em seus perfis permitindo que acessem, por exemplo: conteúdos criados, para facilitar filtros | Isso deve ocorrer para facilitar o dia a dia de quem trabalhará com a plataforma, principalmente professores |
| RF43 | Referência bibliográficas | Média | Preciso adicionar, ao site de forma geral no site, como uma biblioteca e também por conteúdo, questões, etc | Essa adição é importante para quem for criar possa informar de onde tirou essas informações, fornecendo maior credibilidade ao seu conteúdo |
| RF44 | Cálculo de TRI automático | Alta | Adicionar cálculo de TRI automática para simulados, proavs, etc | Conteúdos para isso [INEP TRI](https://www.gov.br/pt-br/noticias/educacao-e-pesquisa/2021/10/entenda-como-e-calculada-a-nota-do-enem) |
| RF45 | Adicionar Feedback aluno/professor nas turmas | Média | Preciso adicionar meio para feedback de entre alunos e professores no site, para facilitar a comunicação | Possíveis formas de fazer isso: forms para inicio, chat privados |
| RF46 | Adicionar chat | Média | Adição de chat em tempo real para turmas (e demais áreas do site) facilitando comunicação nas turmas e também entre professor e aluno| |

## Requisitos não funcionais (RNF)
