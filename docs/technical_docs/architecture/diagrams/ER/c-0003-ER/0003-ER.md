## Atualizações

Nessa nova versão foi pensada para: 
- Permitir deletes confiáveis
- Melhor histórico de criação/updates
- Adição de soft delete
- Atualizado o relacionamento de `system_user` com `skill` e `topic`
- Alteração de UUID para ULID

## Principais Mudanças em Relação à V3
Implementação de Soft Delete
- Antes: Deleções físicas removiam dados permanentemente
- Agora: Campo deleted_at marca registros como deletados
- Benefício: Auditoria completa, recuperação de dados, integridade referencial

## Preparação para Índices e Busca
O modelo está preparado para futuras otimizações:
- Índices em deleted_at para queries eficientes
- Índices compostos: (deleted_at, status) para filtros combinados
- FULLTEXT em campos textuais: title, description, body, statement
- Índices em foreign keys para JOINs otimizados
